# 🩺 AI Medical Chat Assistant

An all-in-one **Streamlit** health companion that combines symptom-based disease prediction, medicine suggestions, a nearby-hospital finder, and a Retrieval-Augmented Generation (RAG) medical chatbot — all in one app.

> ⚠️ **Disclaimer:** This tool provides informational suggestions generated by machine learning models and an AI assistant. It is **not** a substitute for professional medical diagnosis or treatment. Please consult a licensed doctor before making any health decisions.

---

## ✨ Features

| Tab | What it does |
|---|---|
| 🔬 **Disease & Medicine** | Select or type symptoms and get ranked disease predictions with confidence scores, then get medicine suggestions for the predicted (or any known) disease. |
| 🏥 **Nearby Hospitals** | Detects your location via browser GPS or IP, or lets you enter an address/lat-lon manually, then finds and maps nearby hospitals using OpenStreetMap (Nominatim + Overpass) — no API key required. |
| 💬 **Medical Chatbot** | Ask health/medical questions and get answers generated with RAG over a local knowledge base, optionally enriched with live lookups from **PubMed** and **MedlinePlus**. Politely declines non-medical questions. |

## ⚙️ How It Works

- **Disease Prediction** — `models/disease_predictor.py` loads a trained model (e.g. `disease_model.pkl`) and predicts the most likely disease(s) from a list of symptoms, along with confidence scores.
- **Medicine Recommendation** — `models/medicine_recommender.py` loads a trained model (e.g. `madicine_suggestion.pkl`) to suggest medicines for a given disease.
- **Hospital Finder** — `utils/hospital_finder.py` geocodes addresses via **Nominatim** and queries **Overpass API** for nearby hospitals, with an IP-based location fallback.
- **Medical Chatbot (RAG)** — `rag/rag_engine.py` retrieves relevant chunks from a local knowledge base (embedded using `sentence-transformers/all-MiniLM-L6-v2`) and, optionally, live results from PubMed/MedlinePlus, then generates an answer using a configurable LLM provider (OpenAI, Anthropic, or Gemini). If no LLM is configured, it falls back to showing the raw retrieved passages.

## 🚀 Getting Started

### Prerequisites
- Python 3.9+
- pip

### Installation

```bash
git clone https://github.com/maheemax-ux/Ai-Medical-Chat-Assistant.git
cd Ai-Medical-Chat-Assistant
pip install -r requirements.txt
```

### Configuration

The app reads configuration from environment variables or `.streamlit/secrets.toml` (via `config.py`). Set the following as needed:

| Variable | Description | Default |
|---|---|---|
| `LLM_PROVIDER` | `"openai"`, `"anthropic"`, or `"gemini"` | `gemini` |
| `OPENAI_API_KEY` / `OPENAI_MODEL` | OpenAI credentials & model | — / `gpt-4o-mini` |
| `ANTHROPIC_API_KEY` / `ANTHROPIC_MODEL` | Anthropic credentials & model | — / `claude-3-5-sonnet-latest` |
| `GEMINI_API_KEY` / `GEMINI_MODEL` | Gemini credentials & model | — / `gemini-2.5-flash` |
| `ENABLE_WEB_SEARCH` | Enable live PubMed/MedlinePlus lookups | `true` |
| `TAVILY_API_KEY` | Optional, used for web search enrichment | — |

Example `.streamlit/secrets.toml`:

```toml
LLM_PROVIDER = "anthropic"
ANTHROPIC_API_KEY = "your-key-here"
```

### Run the app

```bash
streamlit run app.py
```

The app will open in your browser at `http://localhost:8501`.

## 📦 Key Dependencies

- [Streamlit](https://streamlit.io/) — web app framework

See [`requirements.txt`](requirements.txt) for the full, versioned list.
